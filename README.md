# 产品需求文档 (PRD)：AI 简历与 JD 匹配评估助手 (Web 端)

## 1. 产品概述
* **产品定位：** 一款面向求职者的 AI 驱动 Web 工具，通过上传个人简历与目标岗位 JD，自动输出匹配度体检报告、大白话 JD 翻译以及定制化面试突击指南。
* **核心价值：** 消除招聘话术带来的信息差，量化自身竞争力，提供明确的面试准备方向。

## 2. 核心用户路径 (User Flow)
1. **进入首页** -> 看到左右分布（或上下分布）的两个输入区：左侧「目标职位 JD」，右侧「我的简历」。
2. **输入内容** -> 用户分别通过粘贴文本、上传文件或图片的方式填充两侧数据。
3. **触发解析** -> 点击「开始 AI 匹配分析」按钮，展示极具科技感的 Loading 状态（缓解大模型处理焦虑）。
4. **获取报告** -> 页面平滑过渡到分析结果页，模块化展示：匹配度得分、JD 核心大白话翻译、能力长短板对比、突击建议。
5. **迭代导出** -> 支持将报告导出为长图或 PDF，便于本地复习。

## 3. 功能需求详细说明

### 3.1 输入模块 (Input Zone)
要求支持多模态输入，前端需要做好文件格式校验与大小限制。

* **3.1.1 目标 JD 输入区**
    * **交互形式：** Tab 切换不同的输入方式。
    * **支持方式：**
        * **纯文本粘贴 (主推)：** 提供大文本框，支持富文本格式过滤。
        * **图片上传：** 支持 JPG/PNG，前端或后端调用 OCR（或直接传给多模态大模型如 Gemini 1.5 Pro/GPT-4o）提取文字。
        * **文件上传：** 支持 Word/PDF，大小限制 5MB 内。
        * **网页链接 (高风险保留项)：** 提供输入框，需配合文字提示：“部分招聘平台可能抓取失败，建议直接粘贴文本”。
* **3.1.2 个人简历输入区**
    * **交互形式：** 拖拽上传区域 + 点击上传按钮。
    * **支持方式：** PDF (主推)、Word、图片 (JPG/PNG)。大小限制 10MB 内。

### 3.2 核心 AI 处理模块 (Core Engine)
这是系统的“大脑”，前端获取到文本/图片流后，组装成结构化 Prompt 发送给 LLM。

* **3.2.1 文本提取与清洗 (Pre-processing)**
    * 所有文件和图片输入，在送入大模型匹配前，先统一转化为 Markdown 格式的纯文本。
* **3.2.2 大模型分析调度 (LLM Task)**
    * **任务一：匹配度打分引擎。** 基于简历经历与 JD 要求的重合度，输出一个 0-100 的雷达分数。
    * **任务二：JD “说人话”翻译。** 剥离招聘黑话（如“具备良好的抗压能力”、“赋能业务”），翻译成真实的日常工作内容（如“大概率要经常加班”、“需要你能跨部门要资源推项目”）。
    * **任务三：Gap 分析与突击指南。** 交叉对比后，明确列出“已满足项”和“欠缺项”。针对欠缺项，生成 3-5 个具体的、可被面试官问到的面试题及解答思路。

### 3.3 输出展示模块 (Result Dashboard)
结果页需要极强的结构化和可视化体验。

* **板块一：总体匹配度 (Match Score)**
    * 视觉表现：醒目的环形进度条或仪表盘（例如 85%），配上 AI 简短的一句话定调（如：“匹配度极高，重点准备系统架构设计问题即可”）。
* **板块二：JD 真实意图解析 (JD Translation)**
    * 视觉表现：卡片式左右对比设计。左侧显示“JD 原文摘录”，右侧显示“大白话翻译与潜在潜台词”。
* **板块三：能力雷达与 Gap 分析 (Gap Analysis)**
    * 视觉表现：红绿双色的打勾/打叉列表。
        * 🟢 **核心优势：** 简历中完美契合 JD 的亮点。
        * 🔴 **核心短板/劣势：** 简历中缺失或较弱的部分。
* **板块四：面试突击题库 (Interview Prep)**
    * 视觉表现：折叠面板（Accordion）样式。列出 3-5 道针对短板的定制化面试题，点击展开可看“回答思路与建议话术”。

## 4. 非功能性需求与约束
* **响应延迟兜底：** 大模型处理长文本和图片通常需要 10-30 秒。前端必须设计分步 Loading 动画（如：“正在解析简历...” -> “正在破解 JD 潜台词...” -> “正在生成突击题库...”），严禁在此期间页面卡死。
* **数据隐私保护：** 页面需明确标示：“您的简历仅用于本次单次匹配分析，系统不会存储任何个人隐私数据”。（如果使用 Serverless 架构，确保处理完即销毁缓存）。
* **技术栈建议 (Vibe Coding 友好)：**
    * 前端：Next.js + Tailwind CSS + shadcn/ui (能快速生成极其极客且美观的后台面板界面)。
    * 后端/AI 接入：Next.js API Routes 直接调用大模型 API (建议直接使用支持超大上下文和多模态的模型)。
    * 文件解析：前端使用 `pdf.js` 提取文本，或直接利用模型原生的文件理解能力。

## 快速开始（阶段 0 + 1 + 2 + 3 已实现）

```bash
npm install
npm run dev
```

点击右上角「API Key」配置通义千问 (Qwen) DashScope API Key，Key 仅保存在本地。

### 阶段 2：文件解析（免费模式）

- **PDF**：`pdfjs-dist` 浏览器端解析
- **Word**：`mammoth` 提取纯文本
- **图片**：`tesseract.js` 本地 OCR（中英混合）

### 阶段 3：解析时机与网页链接
- **解析时机**：上传后仅展示文件名、大小、删除按钮，解析在点击「开始 AI 匹配分析」时执行
- **网页链接**：JD 支持输入招聘页 URL，后端抓取并提取文字（部分平台可能失败）

### 大模型与流式输出
- **模型**：通义千问 Qwen (qwen-plus)，接口 `https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions`
- **API Key**：用户输入，保存在 localStorage
- **流式输出**：分析过程流式返回，五步进度展示
